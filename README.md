# Import ERP

ERP-система для управления импортными заказами, производством, документацией и финансовыми расчётами.

## Технологический стек

- **Frontend + Backend:** Next.js 14 (App Router) + TypeScript + Server Actions
- **База данных:** PostgreSQL 16 + Prisma ORM
- **Авторизация:** Auth.js v5 (credentials provider)
- **UI:** TailwindCSS + shadcn/ui
- **Валидация:** Zod + React Hook Form
- **PDF:** @react-pdf/renderer (спецификации, этикетки)
- **Файлы:** локальное хранилище через абстракцию (можно заменить на S3/R2)

## Текущая фаза

**Фаза 2 — Основные бизнес-сущности.** Реализованы:

- Клиенты с несколькими банковскими счетами
- Фабрики (поставщики) с несколькими счетами и валютой контракта
- Товары с фотографиями, размерами (с мерками), вариантами (размер × цвет), сертификатом
- Заказы с позициями, кэшбэком на уровне заказа и автоматическим пересчётом сумм
- История изменений заказа (append-only timeline)
- Архив документов с версионированием — файлы никогда не перезаписываются
- Генерация PDF-спецификации из данных заказа
- Генерация PDF-этикетки (с ЕАС и символами по уходу) из карточки товара

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск PostgreSQL через Docker

```bash
docker compose -f infrastructure/docker-compose.yml up -d
```

### 3. Настройка переменных окружения

```bash
cp .env.example .env
```

Сгенерируйте секрет:

```bash
openssl rand -base64 32
```

Вставьте его в `AUTH_SECRET`.

### 4. Применение миграций и заполнение БД

```bash
npm run db:migrate
npm run db:seed
```

Сидинг создаёт администратора и набор типов документов (Договор, ТН, ТТН, CMR,
Спецификация, Упаковочный лист и т.д.).

### 5. Запуск приложения

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## Конфигурация хранилища файлов

Файлы (фото товаров, сертификаты, документы, PDF) сохраняются в директорию,
указанную в переменной `STORAGE_DIR` (по умолчанию `./uploads`). На Replit
рекомендуется убедиться, что эта директория не очищается между деплоями.

Структура: `{STORAGE_DIR}/{категория}/{id-сущности}/{имя_файла}`.

Замена на S3/Cloudflare R2 / Replit Object Storage в будущем — путём подмены
реализации функций в `src/lib/storage.ts` без изменений в остальном коде.

## Принципы архитектуры

- **Документы** — append-only архив. Новая загрузка = новая версия. Файлы никогда не перезаписываются и не удаляются.
- **Финансы** — Decimal(18, 4), не Float. Никаких потерь точности.
- **История заказов** — каждое изменение пишется в OrderHistory с автором и временной меткой.
- **Кэшбэк** — snapshot **на уровне заказа** (не на уровне фабрики). Каждый заказ хранит свои %/сумму/валюту независимо.
- **Курсы валют** (Фаза 3+) — снимок на момент создания заказа, для воспроизводимости расчётов.
- **Расширяемость** — статусы заказов и типы документов хранятся как config-данные, можно расширять без миграций.
