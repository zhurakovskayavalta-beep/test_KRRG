# Import ERP

ERP-система для управления импортными заказами, производством, документацией и финансовыми расчётами.

## Технологический стек

- **Frontend + Backend:** Next.js 14 (App Router) + TypeScript
- **База данных:** PostgreSQL 16 + Prisma ORM
- **Авторизация:** Auth.js v5 (credentials provider)
- **UI:** TailwindCSS + shadcn/ui
- **Валидация:** Zod + React Hook Form

## Текущая фаза

**Фаза 1 (MVP):** Авторизация и базовая структура проекта.

## Быстрый старт

### 1. Установка зависимостей

```bash
npm install
```

### 2. Запуск PostgreSQL через Docker

```bash
docker compose -f infrastructure/docker-compose.yml up -d
```

### 3. Настройка переменных окружения

```bash
cp .env.example .env
```

Сгенерируйте секрет для авторизации:

```bash
openssl rand -base64 32
```

Вставьте полученное значение в `AUTH_SECRET` в файле `.env`.

### 4. Применение миграций и заполнение БД

```bash
npm run db:migrate
npm run db:seed
```

### 5. Запуск приложения

```bash
npm run dev
```

Приложение доступно на [http://localhost:3000](http://localhost:3000).

## Структура проекта

```
src/
├── app/                  # Next.js App Router (страницы)
│   ├── (auth)/           # Публичные страницы (логин)
│   ├── (dashboard)/      # Защищённые страницы
│   └── api/              # API маршруты (auth handler)
├── modules/              # Бизнес-логика по доменам
│   └── auth/             # Авторизация
├── components/ui/        # UI примитивы (shadcn)
├── lib/                  # Утилиты (prisma, auth, utils)
└── middleware.ts         # Защита маршрутов

prisma/
├── schema.prisma         # Схема БД
└── seed.ts               # Начальные данные

infrastructure/
└── docker-compose.yml    # Postgres для локальной разработки
```

## Доступные команды

| Команда                | Описание                                  |
| ---------------------- | ----------------------------------------- |
| `npm run dev`          | Запуск в режиме разработки                |
| `npm run build`        | Production-сборка                         |
| `npm run start`        | Запуск production-сборки                  |
| `npm run typecheck`    | Проверка типов TypeScript                 |
| `npm run db:migrate`   | Создать и применить миграцию              |
| `npm run db:seed`      | Заполнить БД начальными данными           |
| `npm run db:studio`    | Открыть Prisma Studio                     |
| `npm run db:reset`     | Сбросить БД и применить миграции заново   |
